[ЭТАП 0] Инфраструктура Supabase
─────────────────────────────────
✅ database/schema.sql — создана полная схема PostgreSQL:
     • Таблица cars (external_id UNIQUE, brand, model, year, price, images, specs...)
     • Таблица parts (oem_number, category, stock_qty, price_usd, price_kgs...)
     • Таблица leads (type, name, phone, email, car_id FK, car_info, source_page...)
     • Таблица search_events (page, filters, results_cnt — аналитика поиска)
     • RLS (Row Level Security): публичный SELECT на cars/parts, INSERT на leads
     • Индексы: по brand, region, is_active, year, price
     • Триггеры: auto-update поля updated_at
   Применена в Supabase Dashboard через Management API (скрипт Python)

✅ supabase-client.js — клиент для фронтенда (Vanilla JS, без зависимостей):
     • sbQueryCars(opts) — фильтры: region, brand, model, price, mileage, year, fuel...
     • sbGetBrands(region), sbGetModels(brand, region) — для фильтров
     • sbCountCars(region) — счётчик авто для главной страницы
     • sbSubmitLead(opts) — отправка заявок (замена FormSubmit.co)
     • sbQueryParts(opts) — каталог запчастей
     • sbLogSearch(page, filters, cnt) — аналитика (fire-and-forget)
     • sbLoadCarsWithFallback(region, jsonFile) — Supabase → JSON fallback
     • _normalizeSbCar(row) — алиасы полей для совместимости с HTML-страницами
